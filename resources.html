Books I Recommend

Docs Like Code (Anne Gentle)

What it offers: A practical playbook for treating documentation like software. It shows how version control, branching, reviews, CI, and automated checks make docs faster to ship and easier to maintain. The examples come from real teams moving from wiki or word processor workflows to a docs-as-code pipeline.

What to expect:

  • Setting up a git-based doc workflow and bringing engineers into reviews
  • Static site generators, build pipelines, and checks for links and style
  • Rollout tips for change management and contributor onboarding

Best for: Technical writers, doc leads, and engineers starting a docs-as-code journey.

The Product is Docs (Christopher Gales and the Splunk Documentation Team)

What it offers: Essays from a high-performing docs org on running documentation as a product. It covers hiring, team structure, content strategy, stakeholder management, SLAs, release readiness, quality, and metrics. The tone is candid and operational, which makes it easy to apply in real teams.

What to expect:

  • Frameworks for planning, prioritization, and measuring impact
  • Career ladders, roles, and collaboration models with PM and Engineering
  • Patterns for voice, IA, and release gates that keep docs shippable

Best for: Leads and managers who need scalable practices, and ICs curious about how strong doc orgs operate.
