Books I Recommend

Docs Like Code

Anne Gentle

What it offers: Treat docs like software with git, reviews, CI, and automated checks, so teams ship faster and maintain quality.

  • Git-based doc workflow and engineer reviews
  • Static site generators and CI checks
  • Change management and contributor onboarding

The Product is Docs

Christopher Gales and the Splunk Documentation Team

What it offers: How to run docs as a product with hiring, structure, strategy, SLAs, quality, and metrics that scale.

  • Planning frameworks and impact metrics
  • Roles, ladders, and PM/Engineering collaboration
  • Voice, IA, and release gates for shippable docs

Docs for Developers

Bhatti, Corleissen, Lambourne, Nunez, Waterhouse

What it offers: Practical patterns for developer docs, from doc types to API tutorials, code samples, and review workflows.

  • Structure for tutorials, how-to, concepts, reference
  • Clear code samples and troubleshooting
  • Checklists, contribution models, and feedback loops
